We reuse the temp environment made by TestEnv to avoid trigger recompilation every time.

This can be cleared with clear_testenv_cache().

TestPicker.visualize_test_results — Function

This creates a loop on the latest evaluated test file. It shows a list of the tests that errored and failed with a preview of their stacktrace. You can edit the selected test with Ctrl+e or inspect the stacktrace for errored tests with Enter. It is also possible to inspect the stacktrace as a list with a preview of the source when possible and Ctrl+e edit the source of the current trace.
